An Intro to Algorithms

What is an algorithm?

An algorithm is defined a set or rules or instructions. In computer science, an algorithm is a finite sequence of instructions that a computer executes to perform a specific function. Algorithms can come in many different forms and complete many different tasks, from simple arithmetic to complex pathfinding, as long as a set of instructions is followed, it can be considered an algorithm.

What are some examples of algorithms?

  • Quick Sort

    The quicksort algorithm is a fast sorting algorithm with a time complexity of O(n * log(n)). The algorithm works by choosing a partitions, then comparing the elements within that array to the partition. If the chosen element is greater than the partition, place it to the right, if the element is smaller, place it to the left. With the smaller two unsorted sub arrays, repeat the steps above until the remaining sub arrays are of length 1.

  • Breadth First Search

    Breadth first search is traversal algorithm allowing for the ability search through a given tree or graph. The algorithm searches layer by layer, traversing to a different depth at every step. Starting at an origin node, the algorithm checks all of that nodes neighbors. These new nodes's neighbors are then checked by the algorithm. This process continues until, all the nodes of a given graph or tree are checked. BFS is useful for finding the distance between two nodes in an unweighted directed or undirected graph.

  • Binary Search

    The binary search algorithm is an efficient way to search for a target value within a sorted list. To find a target item using this algorithm, you first select the center item of your sorted list as the value node to compare. Then you compare this item with your target, if your target is greater than the value node, the remaining to the right of the value node becomes the next sub array to check. Otherwise, the next sub array to check is left of the target. These steps are repeated with the remaining sub arrays until the target is found.

Applying algorithms to problems

The following problems apply a variation of the algorithms mentioned above as an optimal solution. Written solutions can be found in Python and in attached resources.

Quick Sort

  • Largest Kth Element In An Array

    Find the Kth largest element of an array, Quick Sort can be used to sort the array before finding the Kth largest element

  • Move Zeros

    Move the zeros to the end of the given array without creating a new array, here the partition value can be set to any non zero element

  • Sort By Parity

    Sort the given array by even or odd, the partition here can be set to the a modulus of the current number then sorted left or right depending on even or odd

  • First Missing Positive (Hard)

    Find the lowest missing positive number, this problem must be solved in linear time and use constant space. Here quick sort can be used with the partition being the correct location of the item in the array. If the current item is positive and in the wrong location, swap it with the element at its correct location. Repeat until the array is sorted, then pass through the array until a missing positive integer is found.

BFS

  • Maximum Depth of Binary Tree

    Given a binary tree, return its maximum depth. Here BFS can be used to check through every item in the tree counting upwards the deeper it searches.

  • Bacteria Colony

    Given a 2d array of bacteria "*", find the number of colonies and return their size and the top left most point. This problem consists of two steps, finding colonies, then finding their size. We can find the top left corner of colonies by starting at (0, 0) and searching leftwards then downwards. If a colony is found, we can use BFS to find its size and track their occupied spaces.

  • Word Ladder (Hard)

    Given a starting word, ending word, and list of valid words. Find the minimum number of letter changes to valid words required to reach the target word. If no word is found, output 0. This problem uses BFS to search through a graph of nodes, where the value of a node is a word, and its edges are possible transformations. We can create a dictionary of the valid words possible states, and store the unmodified possible values at its value. We can use a placeholder letter value "*" to track changed words and compare it with our new modified words. Then we can check all modifications of the previous words possible values.

Binary Search

  • Binary Search

    Locate the target item in a sorted array in O(log(n)) time. This question involves implementing a simple binary search. Use two variables as a low and high pointer, and change their location depending on the range of numbers being searched.

  • First and last position of an element in a sorted array

    In a sorted array, locate the first and last index of a target number, if no number if found, output [-1, -1]. This problem can be solved by including the partition value in the sub arrays. The first index found by including the partition value when it is greater than or equal to the target. The last index found by including the partition value when it is is less than or equal to the target.

  • Capacity To Ship Packages Within D Days

    Given a list of weights and a target number of days. Find the minimum weight capacity of a ship to carry every weight in the target number of days. Here we can use binary search to find the optimal number of days required to ship every weight. First starting at the middle of the sum of every weight, and the weight of the largest element, we can check through every weight tracking the number of days required to ship with this maximum weight. If the number of days is to great, we can increase the maximum weight capacity. If the number of days is to little, we can decrease the maximum weight capacity.

  • Count of smaller numbers after self (Hard)

    Given an array of numbers, output the number of numbers smaller that appear after each item. To solve this problem, we can create a duplicate array of sorted numbers. Then we can loop through the original array and use binary search to find the location of each item within the sorted array. The index of this item, is then the number of smaller elements. We can then remove this element from the sorted array to prevent future searches from adding elements before itself.
