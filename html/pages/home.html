Intro to competitive programming

What is competitive programming?

Competitive programming involves solving a series problems using code faster than other people. But there is more to it than coding. Programmers use their vast knowledge of algorithms and data structures along with their logistical and critical thinking skills to solve the given problems with the most optical solutions.

Big O of n Notation

Machines around the world vary in speed and power, one computer might be able to process a million operations within milliseconds whereas another might take a minute. In order to standardize measuring a programs speed, programmers use a system called big O of n notation O(n). This system compares the input size relative to the number of steps taken to complete. This is known as a programs time complexity.

Finding efficient solutions

When writing programs to solve problems, programmers take into consideration what algorithms and data structures to use based on their time complexities for certain actions. With an effective way to measure the speed, programmers are then able to determine whether a program is fast enough to pass a problem. Although, the choice of what to use may not always be clear.

The Problem

Imagine you are given a array of numbers and a target sum. Two numbers within the array are guaranteed to add up to the target sum without using the index twice. Write a program to output the indices of these two numbers in any order.

For Example:

A list of [2, 7, 11, 15] with a target of 9 would have an output of [0, 1] since the item at index 0 (2) and index 1 (7) add up to 9 (2 + 7 = 9)

A list of [3,2,4] with a target of 6 would have an output of [1, 2] since the item at index 1 (2) and index 2 (4) add up to 6 (2 + 4 = 6)

The Brute Force Approach

The first idea you might have is to compare every number to every other number until you find a sum that reaches the target.

This can be done with two loops looping through the list. The first loop starts at zero and goes through every item:

The brute force approach involves checking every possible combination numbers until you find the solution. Calculating the time complexity of this solution results in O(n^2). This means as the input size gets larger, this program will take exponentially longer to run and output the correct solution.
